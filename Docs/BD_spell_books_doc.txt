База данных проекта spell_book (Книга заклинаний) MySQL

Документация.
База данных содержит следующие таблицы:
-users (Таблица пользователей)
-posts (Основная таблица с записями сайта)
-comments (Таблица с комментариями)
-messages (Таблица личных сообщений)

Структура таблиц:
1. Users
-id (Идентификатор пользователя), тип данных int(11), primary поле, ai автозаполняемое, ключевое.
-nickname (Ник пользователя), тип данных varchar(13) длина поля 13 символов, не нулевое, обязательное.
-name (Имя пользователя), тип данных varchar(20) длина поля 20 символов, не нулевое, обязательное.
-surename (Фамилия пользователя), тип данных varchar(25) длина поля 25 символов, не нулевое, обязательное.
-email (Email пользователя), тип данных varchar(100) длина поля 100 символов, не нулевое, обязательное.
-role (Роль пользователя), тип данных tinyint(4), не нулевое, обязательное. Значение по умолчанию 2 (пользователь). Возможные значения: 0 (администратор), 1 (модератор).
-password (Пароль пользователя/Hash пароля), тип данных varchar(255) длина поля 255 символов, не нулевое, обязательное.
-date_register (Дата регистрации пользователя), тип данных date, не нулевое, обязательное. По умолчанию берет текущую дату (current_timestamp()).
-date_login (Дата последней авторизации пользователя), тип данных date, нулевое, не обязательное.
-avatar (Поле аватара пользователя), тип данных varchar(255) длина поля 255 символов, нулевое, не обязательное.

2. Posts
-id (Идентификатор записи), тип данных int(11), primary поле, ai автозаполняемое, ключевое.
-title (Заголовок записи), тип данных varchar(50) длина поля 25 символов, не нулевое, обязательное.
-text (Текст записи), тип данных text , не нулевое, обязательное.
-date_write (Дата написания), тип данных date, не нулевое, обязательное.
-readings (Прочтения записи), тип данных int(11), Значение по умолчанию 0.
-image (Поле аватара пользователя), тип данных varchar(255) длина поля 255 символов, нулевое, не обязательное.

3. Comments
-id (Идентификатор комментария), тип данных int(11), primary поле, ai автозаполняемое, ключевое.
-post_id (Идентификатор записи), тип данных int(11), нулевое, не обязательное. Для какой записи/ всего сайта комментарий.
-name (Имя пользователя/никнейм), тип данных varchar(20) длина поля 20 символов, не нулевое, обязательное.
-email (Email пользователя), тип данных varchar(100) длина поля 100 символов, не нулевое, обязательное.
-text (Текст комментария), тип данных varchar(255) длина поля 255 символов, не нулевое, обязательное.
-moder_id (Идентификатор модератора), тип данных int(11), нулевое, не обязательное. Кто проверил запись.
-status (Статус записи), тип данных tinyint(4), не нулевое, обязательное. Значение по умолчанию 1 (На проверке). Возможные значения: 0 (Опубликовано), 1 (На проверке), 2(Изъято для всех, кроме администратора).
-date_write (Дата создания комментария), тип данных date, не нулевое, обязательное. По умолчанию берет текущую дату (current_timestamp()).
-date_modification (Дата изменения комментария), тип данных date, нулевое, не обязательное.

4. Messages
-id (Идентификатор сообщения), тип данных int(11), primary поле, ai автозаполняемое, ключевое.
-from_id (Идентификатор пользователя), тип данных int(11), нулевое, не обязательное. Отправитель.
-to_id (Идентификатор пользователя), тип данных int(11), нулевое, не обязательное. Получатель.
-message (Текст сообщения), тип данных varchar(255) длина поля 255 символов, не нулевое, обязательное.
-date_write (Дата создания сообщения), тип данных date, не нулевое, обязательное. По умолчанию берет текущую дату (current_timestamp()).
-date_read (Дата прочтения сообщения), тип данных date, нулевое, не обязательное.

На этом структура базы данных завершена.
